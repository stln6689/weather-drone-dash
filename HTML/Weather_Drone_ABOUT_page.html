Overview

For our EECS 159B Senior Design project, we decided to create a drone that gathers weather data to display on a real-time website, and also make weather predictions (forecasts) based off of previous data models from government websites with our own algorithms.

Objectives

  • Create a drone
  • Have it detect certain weather parameters
  • Display weather conditions on an app/website in real time
  • Create and design the app/website

Formal Summary

A drone will be built with the purpose of measuring multiple weather conditions and updating said conditions real time on a website. The 3 main obstacles within this project will be getting the drone to fly to a high enough altitude to be able to properly measure the weather conditions we want while also maintaining effective (real time) communication with the drone controller and the app, fitting the various types of sensors into our budget, and developing a real time software website that organizes the data measured. The weather condition parameters will be precipitation/humidity, temperature, wind, UV index, and air pollution index.

Use Cases, Customers, Practicality & Application to the Real World

This project’s use case is to detect weather conditions and update the display app/website in real time. The potential customers could be news or broadcasting channels, sport event companies, and finally emergency services.

Process

Fall Quarter

Weeks 1-2 (September 30th - October 11th)

  • As a team, we finalized that we would be building a Weather Drone Forecaster.
  • We created flowcharts with text and visual representation to present our idea to our advisor, Professor Peter Burke.
  • We started the process of researching compatible weather measurement components for the Raspberry Pi that we would be using.
  • We brainstormed the basic concept of the data transmission system.

Week 3 (October 12th - October 18th)

  • We began to finalize the design where the Raspberry Pi would directly connect and transit data to the laptop instead of the drone itself.
  • We started to begin researching which website would work best to display our data. At this point, we had not found a website that would display our data so there were no major developments.

Week 4 (October 19th - October 25th)

  • We started to break down the code for both of our sockets, the Raspberry Pi and the laptop.
  • The battery and receiver were attached to the flight controller. The motors were also attached onto the legs of the drone chastity.

Week 5 (October 26th - November 1st)

  • The python code for the host and server was created, and the server code was able to be executed, which meant the socket could be created. The host client at this time was unable to be connected to, likely due to naming or internet connection issues.
  • The flight controller was attached onto the drone itself.

Week 6 (November 2nd - November 8th)

  • The server and client were able to connect to each other as long as both were connected onto the same website network. Control Flow began to be documented in more detail so that the coding process could be easier.
  • The GPS was installed onto the drone, as well as other positioning aids.

Weeks 7-8 (November 9th - November 22nd)

  • The flight controller on the drone was flashed and GPS was calibrated at this point after some slight issues. The physical parts of the drone were fully completed at this point, which included securing the top frame and velcroing the battery into place.

Weeks 9-10 (November 23rd - December 6th)

  • The final build of the drone was completed in accordance with the EECS 195 class.
  • The Fall Quarter Final Report was created and finalized.

Winter Quarter

Weeks 1-2 (January 6th - January 17th)

  • The server and client code was separated into various segments, making the flow of the code more organized. Testing with the Raspberry Pi had not yet been completed.
  • Telemetry components were purchased; the team was waiting for them to come in to begin implementation.

Week 3 (January 18th - January 24th)

  • Test code was created for the BME280 sensor. The data for temperature, pressure, and humidity was able to be extracted. However, there was a coding bug with the BME280 that needed to be fixed.

Week 4 (January 25th - January 31st)

  • No new progress was made on the code, as the Raspberry Pi could not connect to its server.
  • The Figma Wireframing Draft of our website prototype was created. We began to experiment with the idea of implementing a predictive algorithm for the weather forecast.

Week 5 (February 1st - February 7th)

  • The Raspberry Pi server was fixed after a random reboot. The server was able to begin reading logic to interpret the weather data. Once this was completed, it was able to send it directly to the client’s display.
  • Initial Beta website was created following majority of the skeleton from the Figma Wireframing prototype.

Weeks 6-7 (February 8th - February 21st)

  • Progress was finally made on connecting the UV index telemetry module to the Raspberry Pi. The team realized the module they purchased gave back an analog value, whereas the Raspberry Pi only reads digital values. However after trying a converter module, initial testing the UV index value only came back as the constant 255, or the max value.
  • All of the outline from the Figma prototype has been completed for the website (HTML, CSS)

Week 8 (February 22nd - February 28th)

  • After numerous testings, it was found that the ADP code was outdated for Raspberry Pi 5 instead of Raspberry Pi Zero.
  • Initial Beta website displays temperature, humidity, atmospheric pressure, and UV values, however the JavaScript for live updates and calculating average collected values from data.txt file is still being developed.

Week 9 (March 1st - March 9th)

  • Radio controller having issues connecting with controller.
  • Raspberry Pi Zero and final components have been soldered onto the drone.
  • JavaScript for live updates and calculating average collected values from data.txt file is completed. Final website and formatting has also been finalized and completed.
